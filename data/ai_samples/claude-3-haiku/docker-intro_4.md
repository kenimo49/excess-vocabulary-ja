# Docker入門ガイド

## はじめに

Docker は、アプリケーションの開発、デプロイ、実行を容易にする、非常に人気のある仮想化ツールです。Dockerを使うことで、開発環境から本番環境まで、アプリケーションを一貫して管理することができます。このガイドでは、Docker の基本的な概念や使い方を解説し、Dockerを使ったアプリケーション開発の流れについて説明します。

## Dockerとは

Dockerは、コンテナ型の仮想化技術を提供するオープンソースのプラットフォームです。従来の仮想マシンとは異なり、Dockerはオペレーティングシステムのレベルで仮想化を行います。これにより、アプリケーションとそのランタイム環境をパッケージ化し、移植性の高いコンテナとして扱うことができます。

Dockerの主な特徴は以下の通りです。

- **軽量**: 仮想マシンに比べて、Dockerコンテナはリソース消費が小さい
- **高速**: コンテナの立ち上げ・停止が数秒で完了する
- **柔軟**: アプリケーションを簡単にパッケージ化・配布できる
- **スケーラブル**: コンテナを簡単に複製・拡張できる
- **環境の一貫性**: 開発環境と本番環境の差異を最小限に抑えられる

これらの特徴から、Dockerはアプリケーション開発やデプロイの効率化に大きな効果を発揮します。

## Dockerの基本的な概念

Dockerの主な概念は以下の通りです。

### イメージ

Dockerイメージは、アプリケーションを実行するための土台となるテンプレートです。イメージには、OSの設定、ミドルウェアのインストール、アプリケーションのソースコードなどが含まれています。イメージは、Dockerfileと呼ばれる設定ファイルから構築されます。

### コンテナ

Dockerコンテナは、Dockerイメージをもとに実行されるアプリケーションのインスタンスです。コンテナは、独立した実行環境を提供し、ホストマシンのリソースを共有しながら動作します。

### レジストリ

Dockerレジストリは、Dockerイメージを保管・共有するためのリポジトリです。Docker Hubは、公式の Dockerレジストリサービスです。開発者は、自作のイメージをレジストリにプッシュしたり、公開されているイメージをプルして利用したりすることができます。

### Dockerfile

Dockerfileは、Dockerイメージを構築するための設定ファイルです。Dockerfileには、OSの選択、パッケージのインストール、アプリケーションのコピーなどの手順が記述されています。開発者はこのファイルを編集することで、自分のニーズに合ったイメージを作成できます。

## Dockerの基本的な使い方

Dockerの基本的な使い方は以下の通りです。

### Dockerのインストール

Dockerは、Windows、macOS、Linuxなどの主要なオペレーティングシステム上で利用できます。公式サイトからDockerをダウンロードし、インストーラーに従ってインストールを行います。

### イメージの取得

Dockerイメージは、公開されているレジストリからプルできます。たとえば、Ubuntuの公式イメージを取得するには以下のコマンドを実行します。

```
docker pull ubuntu
```

### コンテナの作成と実行

Dockerイメージからコンテナを作成し、起動することができます。以下のコマンドは、Ubuntuのイメージからコンテナを作成し、bashシェルを起動しています。

```
docker run -it ubuntu bash
```

### Dockerfileによるイメージの構築

アプリケーションに合わせてカスタマイズしたイメージを作成するには、Dockerfileを使います。Dockerfileに必要な設定を記述し、`docker build`コマンドでイメージを構築します。

```
# Dockerfile
FROM ubuntu
RUN apt-get update && apt-get install -y nginx
COPY app/ /var/www/html/
CMD ["nginx", "-g", "daemon off;"]
```

```
docker build -t my-nginx-app .
```

### イメージとコンテナの管理

`docker images`コマンドでイメージの一覧を表示し、`docker ps`コマンドでランニング中のコンテナを確認できます。また、`docker stop`や`docker rm`コマンドを使ってコンテナの停止や削除を行えます。

## Dockerを使ったアプリケーション開発の流れ

Dockerを使ったアプリケーション開発の一般的な流れは以下の通りです。

1. **Dockerfileの作成**: アプリケーションのビルドに必要な手順をDockerfileに記述する
2. **イメージのビルド**: `docker build`コマンドを使ってDockerfileからイメージを構築する
3. **イメージのテスト**: 構築したイメージからコンテナを起動し、アプリケーションが正常に動作することを確認する
4. **イメージのプッシュ**: 必要に応じて、イメージをDockerレジストリにプッシュする
5. **コンテナのデプロイ**: 本番環境でイメージからコンテナを起動し、アプリケーションを稼働させる

Dockerを使うことで、開発環境と本番環境の差異を最小限に抑えながら、アプリケーションのビルドやデプロイを効率的に行えるようになります。
