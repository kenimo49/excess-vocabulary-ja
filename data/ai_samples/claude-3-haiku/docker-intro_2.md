# Docker入門ガイド

## Dockerとは

Dockerは、コンテナ型の仮想化技術を提供するプラットフォームです。従来の仮想マシンとは異なり、Dockerコンテナはオペレーティングシステムのリソースを効率的に共有することができ、高速な起動・停止が可能です。

Dockerのコンテナ型仮想化では、アプリケーションとその実行環境をイメージとして管理します。このイメージを使って、どの環境でも同じ動作をするコンテナをすぐに起動することができます。つまり、「開発環境と本番環境の差異」を解消し、「アプリケーションの移植性」を高めることができるのが大きな特徴です。

## Dockerの基本的な使い方

Dockerを使うにあたって、まずは基本的な使い方を理解しましょう。Dockerには大きく分けて以下の3つの概念があります。

1. **イメージ(Image)**: アプリケーションの実行環境を定義したテンプレート
2. **コンテナ(Container)**: イメージから実際に起動した実行環境
3. **Dockerfile**: イメージを作成するための設定ファイル

### イメージの作成と管理

Dockerイメージは、Dockerfileと呼ばれる設定ファイルから作成します。Dockerfileには、イメージに含めるファイルやソフトウェアのインストール手順などが記述されています。

以下は、Nginxサーバのイメージを作成するためのDockerfileの例です。

```Dockerfile
# ベースイメージとしてUbuntu 20.04を指定
FROM ubuntu:20.04

# Nginxをインストール
RUN apt-get update && apt-get install -y nginx

# Nginxの起動コマンドを設定
CMD ["nginx", "-g", "daemon off;"]

# 80番ポートをホストにマッピング
EXPOSE 80
```

このDockerfileから、`docker build`コマンドを使ってイメージを作成します。

```
$ docker build -t my-nginx .
```

作成したイメージは、`docker images`コマンドで一覧確認できます。

### コンテナの起動と管理

次に、作成したイメージからコンテナを起動します。`docker run`コマンドを使って、以下のように起動できます。

```
$ docker run -d -p 80:80 --name my-nginx-container my-nginx
```

このコマンドでは、`-d`オプションでデタッチドモード(バックグラウンド)で起動し、`-p`オプションでポートのマッピングを行っています。`--name`オプションでコンテナ名を付与しています。

起動したコンテナは、`docker ps`コマンドで一覧確認できます。また、`docker stop`や`docker start`コマンドで停止/起動することができます。

### コンテナ内部の操作

コンテナ内部に入って操作を行うには、`docker exec`コマンドを使います。

```
$ docker exec -it my-nginx-container /bin/bash
```

`-it`オプションでインタラクティブモードで接続し、Bashシェルが起動します。

コンテナ内部では、通常のLinuxコマンドが使えます。例えば、Nginxのログを確認するには以下のようにします。

```
$ docker exec my-nginx-container tail -n 100 /var/log/nginx/access.log
```

### Dockerイメージのレジストリ

Dockerイメージは、DockerHubなどのレジストリに保存・共有することができます。`docker push`コマンドでイメージをレジストリにプッシュし、`docker pull`コマンドでイメージをプルできます。

例えば、自分のDockerHubアカウントにイメージをプッシュする場合は以下のように行います。

```
# イメージにタグ付け
$ docker tag my-nginx your-dockerhub-username/my-nginx

# イメージをプッシュ
$ docker push your-dockerhub-username/my-nginx
```

## Dockerを使った開発ワークフロー

Dockerは、開発・テスト・運用の各フェーズで活用できる強力なツールです。以下に、Dockerを使った開発ワークフローの例を示します。

1. **開発環境の構築**: Dockerfileを使ってアプリケーションの実行環境を定義し、Docker Composeでマルチコンテナ環境を構築する。
2. **テスト環境の準備**: 開発環境と同じイメージを使ってテスト用のコンテナを起動し、自動テストを実行する。
3. **CI/CDパイプラインの構築**: Dockerイメージのビルドとテストを自動化し、本番環境へのデプロイも Docker を使って行う。
4. **マイクロサービスアーキテクチャ**: Dockerコンテナを使ってマイクロサービス化を実現し、スケーラビリティと可用性を高める。
5. **ログ収集とモニタリング**: Dockerコンテナからログを集約し、Prometheusなどのモニタリングツールと連携する。

Dockerを活用することで、開発から運用まで一貫した開発ワークフローを構築できます。
