# Docker入門ガイド

## はじめに

この記事では、Dockerの基本的な概念や使い方について解説します。Dockerは現代のソフトウェア開発において必要不可欠なツールの1つであり、エンジニアにとって知っておくべき知識です。この記事を通して、Dockerの理解を深め、ご自身のプロジェクトでの活用に役立てていただければと思います。

## Dockerとは

Dockerは、コンテナ型の仮想化技術を提供するオープンソースのプラットフォームです。コンテナ型の仮想化とは、OS上で完全に独立したプロセス空間を作り出し、アプリケーションを実行する仕組みのことです。

従来の仮想マシン型の仮想化とは異なり、DockerコンテナはホストOSと共有するカーネルを使用するため、非常に軽量で起動が高速です。これにより、アプリケーションの開発、テスト、デプロイの効率が大幅に向上します。

Dockerの主な特徴は以下の通りです。

- 軽量で高速な起動
- 環境の再現性が高い
- ポータビリティが高い
- スケーラビリティに優れている
- オープンソースで無償利用可能

## Dockerの基本的な使い方

### Dockerイメージの作成

Dockerでアプリケーションを動作させるには、まずDockerイメージを作成する必要があります。Dockerイメージは、アプリケーションを実行するための環境（OS、ミドルウェア、ライブラリなど）を定義したテンプレートです。

Dockerイメージの作成には、Dockerfileと呼ばれる設定ファイルを使用します。Dockerfileには、イメージに含めるファイルやインストールするパッケージ、環境変数の設定などを記述します。

以下は、NginxをインストールしたDockerイメージを作成するDockerfileの例です。

```dockerfile
FROM ubuntu:18.04
RUN apt-get update && apt-get install -y nginx
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

この Dockerfileを使ってイメージを構築するには、以下のコマンドを実行します。

```
docker build -t my-nginx .
```

`-t`オプションでイメージに名前を付けることができます。ビルド完了後、`docker images`コマンドでイメージが生成されたことを確認できます。

### Dockerコンテナの起動

作成したDockerイメージからコンテナを起動するには、以下のコマンドを使用します。

```
docker run -p 8080:80 my-nginx
```

`-p`オプションでホストとコンテナのポートをマッピングしています。これにより、ホストの8080ポートへのアクセスがコンテナの80ポートにルーティングされます。

コンテナ起動後、ブラウザで`http://localhost:8080`にアクセスすると、Nginxのデフォルトページが表示されます。

### コンテナの管理

Dockerには、コンテナの状態を確認したり、操作したりするためのさまざまなコマンドが用意されています。

- `docker ps`: 起動中のコンテナ一覧を表示します。
- `docker stop [コンテナID]`: 指定したコンテナを停止します。
- `docker rm [コンテナID]`: 指定したコンテナを削除します。
- `docker logs [コンテナID]`: 指定したコンテナのログを表示します。
- `docker exec -it [コンテナID] [コマンド]`: 指定したコンテナ内でコマンドを実行します。

これらのコマンドを使って、アプリケーションの開発や運用の過程でコンテナを柔軟に管理することができます。

## DockerHub

DockerHubは、Dockerイメージのレジストリサービスです。ユーザーはDockerHubにイメージをプッシュしたり、公開されているイメージをプルしたりすることができます。

例えば、Ubuntuの最新版のイメージをプルするには、以下のコマンドを実行します。

```
docker pull ubuntu:latest
```

DockerHubには、様々なオープンソースソフトウェアのオフィシャルイメージが公開されています。これらのイメージを活用することで、アプリケーションの構築がより簡単になります。

## Dockerの応用

Dockerは、開発、テスト、本番環境など、ソフトウェアのライフサイクル全般にわたって活用できます。

- 開発環境: 開発者それぞれが同一の環境でアプリケーションを開発できる
- テスト環境: 本番環境と同一の環境でテストが行える
- 本番環境: アプリケーションの安定性と可用性を高められる

また、Dockerを組み合わせてマイクロサービスアーキテクチャを実現したり、Kubernetes等のコンテナオーケストレーションツールと連携させたりすることで、さらに高度な使い方ができます。
