# Docker入門ガイド

## はじめに

Docker は、アプリケーションの開発、配布、実行を簡単にするコンテナ型の仮想化ツールです。
この記事では、Docker の基本的な概念と使い方を解説し、Dockerを使ったアプリケーション開発の入門ガイドとなります。
対象読者はエンジニアを想定しています。

## Dockerとは

Docker は、アプリケーションをコンテナ化して、簡単にデプロイできるようにするツールです。
コンテナとは、アプリケーションとその実行環境をパッケージ化したものです。
コンテナ内では、アプリケーションに必要なライブラリやツール、設定ファイルなどが含まれており、
開発環境と運用環境の違いによる「動作が異なる」問題を解決できます。

Dockerの主な特徴は以下の通りです。

- **軽量**: コンテナはホストOSのカーネルを共有するため、仮想マシンに比べて起動が高速で、リソース消費が少ない
- **モジュール化**: アプリケーションをコンポーネントに分割し、独立して管理・配布できる
- **再現性**: コンテナイメージは環境設定を含むため、開発、テスト、本番で同じ環境を再現できる
- **ポータビリティ**: コンテナは OS やハードウェアに依存せず、どの環境でも動作する

これらの特徴により、Dockerはアプリケーションのビルド、配布、実行を大幅に簡略化できます。

## Dockerの基本概念

Dockerの基本概念は以下の通りです。

### イメージ (Image)
イメージとは、アプリケーションとその実行環境をパッケージ化したものです。
イメージは、ファイルシステムのスナップショットと起動時のコマンドで構成されます。
Dockerは公式のイメージリポジトリ (Docker Hub) から、さまざまなイメージをダウンロードできます。
また、自作のイメージを作成することもできます。

### コンテナ (Container)
コンテナとは、イメージを実行した状態のものです。
コンテナは、隔離された実行環境を提供し、アプリケーションの起動、停止、リソース管理などができます。
1つのイメージから複数のコンテナを起動できます。

### Dockerfile
Dockerfileとは、イメージを自動的に構築するためのスクリプトファイルです。
Dockerfileには、イメージをビルドする手順が記述されています。
これにより、再現性の高いイメージを構築できます。

### Docker daemon
Docker daemonは、Dockerコマンドを受け取り、コンテナの実行などの操作を行うバックグラウンドプロセスです。
Dockerクライアントから、Docker daemonに対してコマンドを送信することで、Dockerの機能を使えるようになります。

## Dockerの基本操作

では、実際にDockerの基本的な操作方法を見ていきましょう。

### Dockerのインストール
まずは、Dockerをインストールする必要があります。
OSに合わせて、公式サイトの手順に従ってインストールしてください。

### イメージの取得
Docker Hubから、必要なイメージをダウンロードします。
たとえば、Nginxのイメージを取得するには、以下のコマンドを実行します。

```
docker pull nginx
```

### コンテナの作成と実行
取得したイメージからコンテナを作成し、起動します。
以下のコマンドは、Nginxのコンテナを作成して起動します。

```
docker run -d -p 80:80 nginx
```

- `-d`: デタッチドモード (バックグラウンド実行)
- `-p 80:80`: ホストの80番ポートとコンテナの80番ポートをマッピング

これで、ホストマシンの80番ポートにアクセスすると、NginxのWebサーバが起動しているはずです。

### コンテナの管理
作成したコンテナの一覧を表示するには、以下のコマンドを使います。

```
docker ps
```

コンテナを停止するには、以下のコマンドを使います。

```
docker stop <コンテナID>
```

### イメージの作成
自作のイメージを作成するには、Dockerfileを使います。
Dockerfileには、イメージをビルドするための手順が記述されています。
以下は、シンプルなDockerfileの例です。

```Dockerfile
FROM nginx:latest
COPY index.html /usr/share/nginx/html/
```

このDockerfileは、Nginxの最新バージョンをベースイメージとし、`index.html`ファイルをコンテナ内のWebサーバのルートディレクトリにコピーするものです。
この Dockerfileからイメージをビルドするには、以下のコマンドを実行します。

```
docker build -t my-nginx .
```

これにより、`my-nginx`というイメージが作成されます。

## Dockerの応用

Dockerは、アプリケーション開発やインフラ構築の際に非常に便利です。
以下では、Dockerの応用例を紹介します。

### マイクロサービスアーキテクチャ
Dockerコンテナは、アプリケーションをモジュール化して管理しやすくします。
各コンポーネントをDockerコンテナ化し、コンテナ間の連携を行うことで、マイクロサービスアーキテクチャを実現できます。

### 開発環境の標準化
Dockerを使って開発環境をコンテナ化することで、開発者間の環境差を解消できます。
Dockerfileを共有することで、同じ環境を簡単に再現できるようになります。

### 継続的インテグレーション (CI) 
Dockerは、アプリケーションのビルドとデプロイを自動化するCIツールと相性が良いです。
Dockerイメージを使ってテストや本番環境の構築を行うことで、信頼性の高いデプロイプロセスを実現できます。

### マルチクラウド環境
Dockerコンテナは、クラウドプロバイダに依存せずに動作するため、マルチクラウド環境の構築に適しています。
同じDockerイメージを使えば、AWS、GCP、Azureなどさまざまなクラウド上でアプリケーションを実行できます。
