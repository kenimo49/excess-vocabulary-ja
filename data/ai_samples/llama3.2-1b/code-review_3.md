# コードレビューの効果的なやり方

**_code_reviewing:  code の品質を高めるために不可欠なスキル_

エンジニアが code を書く際に codeレビューを実施することは、code の品質を高めるために不可欠なスキルです。コードレビューは、code がどのように設計されていますか、どのように機能するかを理解するのに役立ちます。また、code の問題点も見つけやすくなり、改善するのに役立ちます。

**what はcode review?**

code review は、code が正常に機能しているかどうかをチェックすることです。code reviewerは、code の品質を評価し、潜在的な問題点を指摘します。

**code reviewの実際の例**

例えば、以下のようなコードが示されます。

```markdown
// 1.  error_check_1
if (a == 0 && b == 0) {
  return 0;
}

// 2.  error_check_2
if (a == 0 && b == 1) {
  return 0;
}
```

code_reviewの実際の例では、 code reviewerは、条件を組み合わせると、2 つの式が両方とも 0 であると結論付けることができます。これは、 error_check_1 では error_check_2 の条件を超えています。code_reviewの目的は、code が正常に機能しているかどうかを評価し、潜在的な問題点を特定することです。

**code reviewの目的**

code_reviewの主な目的は、code の品質を評価し、潜在的な問題点を特定することです。code reviewerは、code の適切な設計を検討し、潜在的な問題点を検討します。

**code reviewの重要な側面**

code reviewの重要な側面には、以下のことが含まれます。

*   code の品質を評価する
*  潜在的な問題点を特定する
*   code の改善に役立つ

**code reviewの実際の例**

下記の例では、code reviewは実際に実施されました。

```markdown
// 1.  error_check_1
if (a == 0 && b == 0) {
  return 0;
}

// 2.  error_check_2
if (a == 0 && b == 1) {
  return 0;
}

// 3.  code_review_result
// code_review結果: code_1がnormalに機能している
```

この例では、code_reviewは実際に実施され、code_reviewの結果はcode_1が正常に機能しているという結論が得られました。
