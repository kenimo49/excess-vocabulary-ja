# REST API 設計のベストプラクティス

REST (Representational State of Resource) API 設計は、Web API 設計における重要な概念です。REST API 設計における最も重要な特徴は、HTTP です。HTTP では、リソースをリクエストすると、リソースのベースに基づいて返されたデータを返します。

## HTTP  Status Code の使用

HTTP では、異なるステータコードが使用されます。例えば、200のステータコードは、リソースが正常に返された場合に使用されます。500のステータコードは、システムエラーを表します。

### HTTP  Status Code の例

*   201 Created: このステータコードは、新しいリソースを表示します。
*   404 Not Found: このステータコードは、指定されたリソースが見つかりませんでした。
*   409 Conflict: このステータコードは、存在しないリソースに変更が実施されました。

## HTTP Method の使用

HTTP では、異なるアプローチを使用できます。主なアプローチは、GET、POST、PUT、DELETEです。

*   GET: HTTP 1.1 では GET では、リソースの基本的な情報を表示します。GET では、リソースのベースに基づいて返されたデータを返します。
*   POST: HTTP 1.1 では POST では、新しいリソースを表示します。
*   PUT: HTTP 1.1 では PUT では、リソースのベースに基づいて返されたデータを更新します。
*   DELETE: HTTP 1.1 では DELETE では、リソースを削除します。

### HTTP Method の例

*   GET: `/users/1` で表示されます。
*   POST: `/users` で新しいリソースを生成します。
*   PUT: `/users/1` でリソースのベースに基づいて返されたデータを更新します。
*   DELETE: `/users/1` でリソースを削除します。
