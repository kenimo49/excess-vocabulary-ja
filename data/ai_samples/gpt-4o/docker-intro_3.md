# Docker入門ガイド

こんにちは、エンジニアの皆さん。今回のブログ記事では、Dockerについての基礎知識とその利点、使い方を解説します。Dockerは近年ますます注目を集めるコンテナ技術で、多くの開発現場で利用されています。Dockerの魅力は、アプリケーションの実行環境を一つのパッケージにまとめることで、環境依存の問題を解決し、開発・運用を大幅に簡素化する点にあります。

## Dockerとは？

Dockerはコンテナ技術の一種で、アプリケーションとその実行環境を一つの単位として「コンテナ」にパッケージ化するツールです。Dockerは軽量でポータブルな特性を持ち、どこでも同じ環境でアプリケーションを実行できます。これにより、開発環境と本番環境の差異を最小限にし、「動いたけど、なぜか動かない」といった問題を解消します。

## なぜDockerを使うべきか？

### 1. 一貫性のある環境

Dockerを使用することで、開発環境、本番環境、テスト環境などが一貫した状態で保たれます。異なる環境間での不整合が減少し、「自分のマシンでは動くのに…」という状況を避けられます。

### 2. スケーラビリティ

Dockerコンテナは必要に応じて簡単に増やしたり減らしたりすることができます。これにより、急なトラフィックの増加にも柔軟に対応可能です。

### 3. 簡単なデプロイ

Dockerイメージを作成すれば、そのイメージを使ってどの環境にも同じアプリケーションをデプロイ可能です。デプロイが非常に簡単になり、CI/CDパイプラインにも容易に統合できます。

### 4. 効率的なリソース使用

従来の仮想マシンと比べて、Dockerコンテナはホストのカーネルを共有するため、リソースのオーバーヘッドが小さくなります。これにより、より多くのコンテナを一つのホスト上で稼働させることが可能です。

## Dockerの基本概念

### イメージとコンテナ

- **イメージ**: 実行可能なアプリケーションを構成するファイルとその設定のスナップショットです。イメージは不変で、コンテナのテンプレートとして使用されます。
  
- **コンテナ**: イメージをもとにして実行されるプロセスです。コンテナは一時的なもので、必要に応じて起動・停止・削除できます。

### Dockerfile

Dockerfileは、イメージを構築するためのレシピです。どのベースイメージを使用するか、どのコマンドを実行するか、ファイルのコピー、環境変数の設定などを記述します。

```Dockerfile
# ベースイメージを指定
FROM node:14

# 作業ディレクトリを設定
WORKDIR /app

# ローカルファイルをコンテナ内にコピー
COPY . .

# 依存関係をインストール
RUN npm install

# アプリケーションを実行
CMD ["npm", "start"]
```

### Docker Hub

Docker Hubはコンテナイメージのリポジトリサービスです。ここからさまざまなイメージを取得したり、自身のイメージを公開して他の開発者と共有できます。

## Dockerの基本操作

### インストール

Dockerのインストールは公式サイトからプラットフォームに応じたインストーラをダウンロード・実行するだけで済みます。インストール後、以下のコマンドで動作確認を行います。

```bash
docker --version
```

### Dockerの基本コマンド

#### Dockerイメージの取得

```bash
docker pull ubuntu:latest
```

#### Dockerコンテナの起動

```bash
docker run -d -p 80:80 nginx
```

#### コンテナのリスト表示

```bash
docker ps
```

#### コンテナの停止

```bash
docker stop [コンテナID]
```

#### コンテナの削除

```bash
docker rm [コンテナID]
```

#### イメージの作成

```bash
docker build -t my-node-app .
```
