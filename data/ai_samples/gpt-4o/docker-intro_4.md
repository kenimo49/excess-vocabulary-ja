# Docker入門ガイド

現代のソフトウェア開発において、Dockerは欠かせないツールの一つとなっています。コンテナ技術を利用することにより、開発環境と本番環境の差異を減らし、アプリケーションの移植性とスケーラビリティを向上させることができます。本記事では、Dockerの基本的な概念から、実践的な使用方法までを解説します。

## 1. Dockerとは？

Dockerは、アプリケーションをコンテナという軽量かつ独立した実行環境にパッケージングするためのオープンプラットフォームです。これにより、開発および配布が簡略化され、環境間での互換性の問題を軽減します。

### コンテナとは？

コンテナは、アプリケーションとその動作に必要なライブラリや設定をまとめたパッケージです。これにより、開発した環境と本番環境での動作を一貫させることができます。コンテナはホストOS上で動作するため、仮想マシンのようにOS全体を模倣する必要がなく、より高速かつ軽量です。

## 2. Dockerの基本用語

Dockerを理解するためには、以下の基本用語を押さえておくことが重要です。

- **イメージ (Image)**: コンテナを作成するためのテンプレート。アプリケーションや依存関係を含んでおり、Docker Hubなどのリポジトリから取得することができます。
- **コンテナ (Container)**: 実際に実行されるインスタンス。イメージを基に作成され、アプリケーションを動作させます。
- **Dockerfile**: イメージを作成するための設定ファイル。どのような環境やアプリケーションを含むかが記述されます。
- **Docker Hub**: Dockerイメージのレジストリサービス。多くの公式イメージが公開されています。

## 3. Dockerのインストール

始めに、Dockerを自分のマシンにインストールする必要があります。DockerはWindows、macOS、Linuxに対応していますが、各OSごとのインストール手順は公式サイトの[インストールガイド](https://docs.docker.com/get-docker/)を参照してください。

## 4. Dockerの基本コマンド

Dockerを使い始めるために、いくつかの基本的なコマンドを紹介します。

### イメージの取得

Docker Hubからイメージを取得するためには、以下のコマンドを使用します。

```bash
docker pull イメージ名
```

たとえば、公式のnginxイメージを取得する場合は次のようにします。

```bash
docker pull nginx
```

### コンテナの作成と実行

取得したイメージからコンテナを作成して実行するには、以下のコマンドを使用します。

```bash
docker run -d --name=mynginx -p 8080:80 nginx
```

このコマンドは、`nginx`イメージからバックグラウンドで実行されるコンテナを作成し、ホストのポート8080でリッスンするようマッピングします。

### コンテナの確認

現在実行中のコンテナを確認するには、以下のコマンドを使用します。

```bash
docker ps
```

停止中のコンテナも含めて全てのコンテナを確認する場合は、次のコマンドを使用します。

```bash
docker ps -a
```

### コンテナの停止と削除

動作中のコンテナを停止するには以下のコマンドを使用します。

```bash
docker stop コンテナID
```

コンテナを削除するには以下のコマンドになります。

```bash
docker rm コンテナID
```

## 5. Dockerfileの作成

Dockerfileを使用することでオリジナルのDockerイメージを作成することができます。以下に簡単なNode.jsアプリケーションのDockerfileの例を紹介します。

### Dockerfileの例

```dockerfile
# ベースイメージとして公式のNode.jsイメージを使用
FROM node:14

# 作業ディレクトリを設定
WORKDIR /app

# アプリケーションのソースをコンテナにコピー
COPY . .

# 依存関係をインストール
RUN npm install

# コンテナが実行されたときに実行されるコマンド
CMD ["node", "app.js"]
```

この例では、Node.jsのベースイメージを使用し、アプリケーションのソースコードと依存関係をコンテナにコピーしています。そして、`npm install`を実行し、`node app.js`でアプリケーションを起動します。

### イメージのビルド

Dockerfileからイメージをビルドするには、次のコマンドを使用します。

```bash
docker build -t mynodeapp .
```

`-t`オプションでタグを指定し、ビルドされたイメージを識別しやすくします。

## 6. Docker Compose

Docker Composeを使うことで、複数のコンテナをまとめて管理することができます。`docker-compose.yml`ファイルにコンテナの構成を記述し、一括で操作できます。

### docker-compose.ymlの例

以下は、簡単なDocker Composeの設定ファイルの例です。

```yaml
version: '3'
services:
  web:
    image: nginx
    ports:
      - "8080:80"
  app:
    build: .
    ports:
      - "3000:3000"
    volumes:
      - .:/app
```

このファイルは、`nginx`とそれに依存するアプリケーションコンテナを設定します。

### Docker Composeの起動

Docker Composeの設定に基づいてコンテナを起動するには、以下のコマンドを使用します。

```bash
docker-compose up
```

`-d`オプションをつけることでバックグラウンドで起動することも可能です。

## まとめ

Dockerはアプリケーションの開発、配布、実行を効率化する強力なツールです。この記事では、Dockerの基本的な概念と基本コマンド、そしてDockerfileを使ったカスタムイメージの作成方法とDocker Composeを利用した複数コンテナの管理について説明しました。実際に手を動かしながら、Dockerの理解を深めていただけると幸いです。Dockerを活用して、よりシンプルで効率的な開発環境を構築していきましょう。
