import pandas as pd
import matplotlib.pyplot as plt
def python_function(df: pd.DataFrame) -> str:
    # 日付列をdatetime型に変換
    df['日付'] = pd.to_datetime(df['日付'])
    # 2010年以降のデータを抽出
    df = df[df['日付'].dt.year >= 2010]

    # 平均価格をグラフ化
    df_grouped.plot(kind='line', title='年平均為替価格（2010年以降）')

    plt.legend(title='通貨')
    # グラフをPNGファイルとして保存
    plt.savefig('temp/output.png')
    # 処理したデータフレームをExcelファイルとして保存
    with pd.ExcelWriter('temp/output.xlsx', engine='xlsxwriter') as writer:

    """

#### PandasのDataframeでの日付型の取り扱い
PandasのDataframeでの日付型が、デフォルトだと想定どおりにCastされてくれないため、補正してやらないといけない。
- CSVからPandasのDataframeに値をセットした場合、日付型でセットされてほしい項目もString型でセットされる。そのため、以下のようなコードで日付型にCastする。
- 型を正確にGPT-4に伝えることは、コード生成において重要事項であるため、適切なCastを行う必要がある。

■ 次のコードで行っているCast
- テキストデータが、日付型に適した文字列だったら、Time型かDatetime型にキャストする。
- Date型(日付のみの型)はOpenaiが生成したコードでエラーになりやすいため、Datatime型にする。
```python
def optimize_df_types(df:pd.DataFrame) -> pd.DataFrame:
    # str型の項目に対して日付型に変換できるかどうかをチェックし、変換する
    for col in df.select_dtypes(include=['object']).columns:
        try:
            if ':' in df[col].iloc[0]:
                if len(df[col].iloc[0]) == 5:  # 5桁の場合は秒を補完する
                    df[col] = df[col].apply(lambda x: str(x) + ':00' if len(str(x)) == 5 else x)
                    df[col] = pd.to_datetime(df[col]).dt.time
                    print(f'{col}: Converted to time')
                elif len(df[col].iloc[0]) == 8:
                    df[col] = df[col].apply(lambda x: str(x) if len(str(x)) == 8 else x)
                    df[col] = pd.to_datetime(df[col]).dt.time
                    print(f'{col}: Converted to time')
                elif '/' in df[col].iloc[0] or '-' in df[col].iloc[0]:
                    df[col] = pd.to_datetime(df[col])
                    print(f'{col}: Converted to datetime')
                else:
                    print(f'{col}: Should not converted to datetime')
            else:
                this_year:str = datetime.now().strftime("%Y")
                if '/' in df[col].iloc[0]:
                    df[col] = df[col].apply(lambda x: this_year + '/' + str(x) if len(str(x)) == 5 else x) # 5桁の場合は年を補完する
                    df[col] = pd.to_datetime(df[col])
                    print(f'{col}: Converted to date')
                elif '-' in df[col].iloc[0]:
                    df[col] = df[col].apply(lambda x: this_year + '-' + str(x) if len(str(x)) == 5 else x) # 5桁の場合は年を補完する
                    df[col] = pd.to_datetime(df[col])
                    print(f'{col}: Converted to date')
                else:
                    print(f'{col}: Should not converted to datetime')
        except ValueError:
            print(f'{col}: Cannot be converted')
    return df
```

    - 自前開発の利点は、カスタマイズ性が高い点。ただし、多機能すぎる要求はスクラッチ開発現場の開発スキルやコストでは難しいと考えられるため、特定のシナリオに特化し、開発難易度を下げる必要がある
- 画面での回答遅い
    - まずGPT-4の回答が遅い。プロンプトが複雑になるため、おおむねAOAIで20～60秒、OpenAI本家で40～90秒かかる ※2023年10月時点の計測。今後大きく変わる可能性は十分あり。
    - 実装によっては複数回のGPT-4呼び出しが必要になり、回数分だけ待ち時間が増える
- 処理内容に対しての信頼度をどうあげるか。利用者の意図どおりの加工になっているか。
    - 前章のように、可能な限りの処理説明は付与するが、信頼度を上げるにはPythonのコードを読むしかない
    - Excelのようにコード理解のスキルが利用者側に広がるまでは、時間がかかるので懸念がある
- Python実行環境のパフォーマンスへの考慮
    - 他の処理に比べて、Pythonによる加工処理は負荷が大きいため、それを受け付ける仕組みやハードウェアリソースが必要
- CSVアップロードにする場合の課題
    - CSVのフォーマットは利用者に委ねることになる。例えば、項目名やプロンプトに「金額」という言葉がないが、依頼文に「売上金額を教えて」と聞いてしまう人がたくさんいると考えられる。GTP-4はある程度推測して答えてくれるが、精度はインプットデータの文脈なので、利用者にもリテラシーが求められてしまう
- 本番化の世の中的な実績が浅い
    - 歴史の浅い機能のため、本番化に向けておよび本番運用において、技術課題が発生する可能性が高い。都度対処していく必要がある

### 2-6-8. Excelを編集するマクロ的なCode Interpreter

- 以下2通りの方法でExcelのセルを編集したりコピーしたりするCode生成を行い実行させました。
    - Excelのマクロとして実行するVBAコードの生成
    - Excelのセルを編集するPythonコードの生成

どちらとも実用に耐える制度でのコード生成はできませんでした。
5回に1回くらい、ほぼほぼ正解のコードを書く程度の精度が限界でした。
原因は、おそらく学習データの中にあるサンプルコードが世の中にほとんど公開されてないからだと考えられます。残念な結果でした。
