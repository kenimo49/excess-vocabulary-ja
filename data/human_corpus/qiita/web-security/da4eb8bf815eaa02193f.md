# ２．IPAとは
情報処理試験の運営者としてなじみのある機関ですが、どう定義されるのでしょうか。
<a herf="https://ja.wikipedia.org/wiki/%E6%83%85%E5%A0%B1%E5%87%A6%E7%90%86%E6%8E%A8%E9%80%B2%E6%A9%9F%E6%A7%8B">wikipedia</a>によると、
>独立行政法人情報処理推進機構（じょうほうしょりすいしんきこう、英: Information-technology Promotion Agency, Japan、略称: IPA）は、日本のIT国家戦略を技術面・人材面から支えるために設立された独立行政法人（中期目標管理法人）。所管官庁は経済産業省。

:point_up:情報処理試験では「経済産業省が優れている」という選択肢を選ぶと正解するといううわさもあったりします。

ちなみに「セキュリティ」を身近に学んでもらうためか、中の人の趣味かはわかりませんが、だいたい二次元キャラで啓蒙活動が行われています。

# ３．セキュリティ10大脅威とは

<a herf="https://www.ipa.go.jp/security/vuln/10threats2022.html">IPAのサイト</a>によると、

>「情報セキュリティ10大脅威 2022」は、2021年に発生した社会的に影響が大きかったと考えられる情報セキュリティにおける事案から、IPAが脅威候補を選出し、情報セキュリティ分野の研究者、企業の実務担当者など約150名のメンバーからなる「10大脅威選考会」が脅威候補に対して審議・投票を行い、決定したものです。

毎年、8月下旬に発表されます。

2022年はこんな感じでした。

# ４．2022年のセキュリティ10大脅威(組織の部)

IPAのレポートを読みながら、「組織の部」について整理してみました。

||名称|特徴|感染経路|予防策|
|:-----------|:-----------|:-----------|:------------|:------------|
|1|ランサムウェア|データを暗号化し、復旧することと引き換えに金銭を要求する|・メール(でクリック、添付ファイル開封)<br>・webサイトクリック<br>・ソフトウェア脆弱性を突く|・CSIRT<br>・多要素認証<br>・従業員教育(不明なURLはクリックしない)・バックアップの取得＆復旧訓練・迅速なネットワーク分離<br>・ソフトウェア脆弱性対応|

|5|内部不正による情報漏洩|従業員/元従業員の機密情報持ち出し|・アクセス権限悪用<br>・内部情報不正持ち出し|・従業員教育<br>・アカウント管理<br>・アクセスログ監視<br>・不正/予兆検知|
|6|脆弱性対策情報の公開に伴う悪用増加|公開された脆弱性情報を利用し、未対応製品を攻撃|・Nデイ脆弱性を悪用<br>・公開されている攻撃ツールを使用|・ネットワーク監視/攻撃通信遮断<br>・一時的にサーバ停止<br>・攻撃予兆検知(UTM,IDS/IPS,WAFなど)<br>・脆弱性収集と対応の徹底(予算確保)|
|7|修正プログラムの公開前を狙う攻撃(ゼロデイ攻撃)|ソフトウェアの脆弱性対応パッチの公開前に攻撃|ソフトウェア、ハードウェアの脆弱性|・ネットワーク監視/攻撃通信遮断<br>・セキュリティのサポートが充実しているソフトウェアやバージョンを使う。<br>・回避策/緩和策を迅速に講じる<br>・攻撃予兆検知(UTM,IDS/IPS,WAFなど)<br>・脆弱性収集と対応の徹底(予算確保)|
|8|ビジネスメール詐欺による金銭被害|巧妙な騙しの手口を駆使したメールを送りつけ、従業員を騙し誤って送金させる(なりすましなど)|・メール|・メールに依存しない業務フロー構築<br>・メールに電子署名付与(S/MIMEやPGP)<br>・電話など複数経路で事実確認・従業員教育(メールの差出人ドメインの確認、普段と異なる表現)|
|9|予期せぬIT基盤の障害に伴う業務停止|クラウドサービス停止により業務が止まることで、サービス提供ができなくなる(医療など)|(発生要因として)<br>・自然災害<br>・作業事故<br>・設備障害<br>・システム障害(ネットワーク輻輳など)|・予備システムの用意(DRサイト)<br>・被害訓練|
|10|不注意による情報漏洩等の被害|本人の不注意による行動で情報漏洩が発生しさらに悪用される(PC置き忘れ、メール誤送信)|(発生要因として)<br>・本人が何らかの要因で注意力低下<br>・意識低下|・従業員教育<br>・持ち出し禁止ルール<br>|

全体を通して「必要な対策」とは、今年に限ったものではなく、
・CSERTのような組織を確立する
・従業員教育
・被害を受けた後の行動確認、被害を最小に防ぐ策をあらかじめ講じる
・日頃の脆弱性情報収集と対応の徹底

あたりかなと思います。
これまでセキュリティ対策の投資は景気に左右されがちでしたが、コンプライアンスの重要性が増している今、本腰を入れる企業も多いのではないでしょうか。
