# 【物流AI #17】Sim-to-Real：シミュレータで学習したロボットを実機に移す

## Sim-to-Real ギャップとは

シミュレータと現実の差（摩擦・レイテンシ・センサーノイズ）のため、シミュレータで完璧に動いても実機では失敗する。これを **Sim-to-Real ギャップ** と呼ぶ。

## Domain Randomization（ドメインランダム化）

シミュレータのパラメータをランダムに変動させて学習し、実機の多様な条件を事前に「経験」させる。

```python
import numpy as np

class RandomizedPickEnv:
    def __init__(self):
        self.reset()

    def randomize(self):
        self.mass     = np.random.uniform(0.1, 2.0)    # kg
        self.friction = np.random.uniform(0.3, 0.9)
        self.latency  = np.random.uniform(0.01, 0.05)  # sec
        # カメラノイズ
        self.cam_std  = np.random.uniform(0.0, 0.02)
        # アクチュエータ誤差
        self.act_noise = np.random.uniform(0.0, 0.05)

    def reset(self):
        self.randomize()
        return self._obs()

    def step(self, action):
        # アクチュエータノイズを加算
        noisy = action + np.random.normal(0, self.act_noise,
                                          action.shape)
        # 物理ステップ（MuJoCo 等を想定）
        obs, rew, done, info = self._physics_step(noisy)
        # カメラ観測にノイズ
        obs[:3] += np.random.normal(0, self.cam_std, 3)
        return obs, rew, done, info
```

## Adaptive Domain Randomization

固定範囲ではなく、**実機データとのずれ**を見て範囲を自動調整する。

```python
def update_dr_range(sim_traj, real_traj, param_range,
                    lr=0.1, target_div=0.1):
    div = js_divergence(sim_traj, real_traj)  # JS 情報量
    if div > target_div:
        # ランダム化を弱める（実機に近づける）
        param_range["std"] *= (1 - lr)
    else:
        # ランダム化を強める（汎化を高める）
        param_range["std"] *= (1 + lr * 0.5)
    return param_range
```
