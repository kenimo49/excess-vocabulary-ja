

CloudWatch Logs のメトリクスフィルターやログのフィルタリングでは、**フィルターパターン**を使ってログイベントを絞り込みます。パターンの書き方を正しく理解することで、必要なログだけを確実にキャッチできます。

---

## 1. 基本構文

### スペース区切り = AND 条件

複数の語をスペースで区切ると、**すべてを含む（AND）** ログにマッチします。

```
ERROR Database
```
→ `"ERROR"` かつ `"Database"` を含むログにマッチ

### `?` プレフィックス = OR 条件

各語の前に `?` を付けてスペースで区切ると、**いずれかを含む（OR）** ログにマッチします。

```
?ERROR ?error ?Error ?エラー
```
→ `"ERROR"`、`"error"`、`"Error"`、`"エラー"` のどれかを含むログにマッチ

> ⚠️ **よくある間違い：** `?Error?エラー` のようにスペースなしで書くと、`Error?エラー` という1つのトークンとして解釈されます。**必ずスペースで区切ってください。**

### `-`（マイナス）プレフィックス = 除外条件

特定の語を含むログを除外します。

```
ERROR -DEBUG
```
→ `"ERROR"` を含み、かつ `"DEBUG"` を含まないログにマッチ

---

## 2. 大文字・小文字の区別

CloudWatch Logs のフィルターパターンは**大文字・小文字を区別します（case-sensitive）**。

```
# 以下はそれぞれ別のパターン
ERROR   → "ERROR" のみにマッチ
error   → "error" のみにマッチ
Error   → "Error" のみにマッチ
```

大文字・小文字を問わずマッチさせたい場合は、`?` を使って OR 条件で列挙します。

```
?ERROR ?error ?Error
```

---

## 3. クォートを使ったフレーズ検索

スペースを含むフレーズを検索する場合は、**ダブルクォート `"..."` で囲みます**。

```
"Internal Server Error"
```
→ このフレーズをそのまま含むログにマッチ（スペース込みで1つのトークン扱い）

```
"NullPointerException" "at com.example"
```
→ 両方のフレーズを含むログにマッチ（AND 条件）

---

## 4. ワイルドカード `*`

`*` は任意の文字列にマッチします。

```
*Exception
```
→ `NullPointerException`、`IllegalArgumentException` など末尾が `Exception` のもの全てにマッチ

```
"User*login failed"
```
→ `"User123 login failed"` や `"UserAdmin login failed"` にマッチ

---

## 5. JSON ログのフィルタリング

ログが JSON 形式の場合、`{ }` を使って特定のフィールド値でフィルタリングできます。

### 基本構文

```
{ $.フィールド名 = "値" }
```

### 比較演算子

| 演算子 | 意味 | 例 |
|--------|------|----|
| `=` | 等しい | `{ $.level = "ERROR" }` |
| `!=` | 等しくない | `{ $.level != "INFO" }` |
| `>` | より大きい | `{ $.statusCode > 499 }` |
| `>=` | 以上 | `{ $.statusCode >= 500 }` |
| `<` | より小さい | `{ $.responseTime < 100 }` |
| `<=` | 以下 | `{ $.responseTime <= 3000 }` |
| `EXISTS` | フィールドが存在する | `{ $.errorCode EXISTS }` |
| `NOT EXISTS` | フィールドが存在しない | `{ $.errorCode NOT EXISTS }` |

### AND / OR の組み合わせ

```
{ $.level = "ERROR" && $.statusCode >= 500 }
```
→ level が ERROR かつ statusCode が 500 以上

```
{ $.level = "ERROR" || $.level = "FATAL" }
```
→ level が ERROR または FATAL

### ネストされたフィールド

```
{ $.user.role = "admin" }
```

### 配列フィールド

```
{ $.errors[0].code = "TIMEOUT" }
```

---

## 6. よく使うパターン集

### エラー系ログを全部キャッチ

```
?ERROR ?error ?Error ?FATAL ?fatal ?Fatal ?エラー
```

### HTTP 5xx エラー（JSON ログ）

```
{ $.statusCode >= 500 }
```

### レスポンスタイムが遅いリクエスト（3秒以上）

```
{ $.responseTime >= 3000 }
```

### 特定ユーザーの操作ログ

```
{ $.userId = "user-12345" }
```

### Exception を含むがテスト系を除外

```
Exception -Test -test -Mock -mock
```

### 認証失敗ログ

```
"authentication failed" ?unauthorized ?Unauthorized ?"401"
```

---

## 7. スペースなしテキストのフィルタリング

ログ出力がスペースなしで連結されている場合（例：`[ERROR]2024-01-01`）も、部分文字列としてマッチします。

```
[ERROR]
```
→ ログに `[ERROR]` が含まれていればマッチ（前後にスペース不要）

---

## 8. メトリクスフィルターでの使用例

### 設定例：5xx エラー数を監視（JSON ログ）

| 項目 | 設定値 |
|------|--------|
| フィルターパターン | `{ $.statusCode >= 500 }` |
| メトリクス名前空間 | `MyApp/API` |
| メトリクス名 | `5xxErrorCount` |
| メトリクス値 | `1` |
| デフォルト値 | `0` |

### 設定例：エラーログ数を監視（テキストログ）

| 項目 | 設定値 |
|------|--------|
| フィルターパターン | `?ERROR ?error ?Error ?エラー` |
| メトリクス名前空間 | `MyApp/Application` |
| メトリクス名 | `ErrorLogCount` |
| メトリクス値 | `1` |
| デフォルト値 | `0` |

## 9. デバッグのコツ

### CloudWatch コンソールでパターンをテスト

1. CloudWatch → ロググループ → 対象グループを選択
2. **「メトリクスフィルター」** タブ → **「メトリクスフィルターの作成」**
3. **「パターンのテスト」** セクションにログ文字列を貼り付けて即確認

### チェックリスト

- [ ] OR 条件の `?term` の間にスペースが入っているか
- [ ] 大文字・小文字のバリエーションを網羅しているか
- [ ] JSON フィルターは `{ }` で囲まれているか
- [ ] JSON のフィールドパスは `$.` で始まっているか
- [ ] フレーズ検索はダブルクォートで囲まれているか

---

## 10. よくある間違いまとめ

| ❌ 誤り | ✅ 正しい | 理由 |
|---------|---------|------|
| `?Error?エラー` | `?Error ?エラー` | スペースがないと1トークンになる |
| `{$.level = ERROR}` | `{ $.level = "ERROR" }` | 文字列値はダブルクォートが必要 |
| `$.level = "error" OR $.level = "ERROR"` | `{ $.level = "error" \|\| $.level = "ERROR" }` | JSON フィルターは `{ }` 内で `\|\|` を使う |
| `*` のみ | （使用不可）| 単独の `*` はパターンとして無効 |
| `ERROR error` | `?ERROR ?error` | スペース区切りは AND 条件になる |

---

## まとめ

| 条件 | 書き方 | 例 |
|------|--------|-----|
| AND（かつ） | スペース区切り | `ERROR Database` |
| OR（または） | `?` プレフィックス + スペース | `?ERROR ?error ?エラー` |
| 除外（含まない） | `-` プレフィックス | `ERROR -DEBUG` |
| フレーズ | ダブルクォート | `"login failed"` |
| ワイルドカード | `*` | `*Exception` |
| JSON フィールド | `{ $.field = "value" }` | `{ $.level = "ERROR" }` |
