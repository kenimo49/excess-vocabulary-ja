# 【物流AI #10】グラフニューラルネットワークで動的配送経路を学習する

## GNN を配送最適化に使う理由

VRP（配送計画問題）を GNN でモデル化すると、「顧客グラフの構造」を特徴として学習できる。新規インスタンスへの汎化が QAOA より速く、リアルタイム用途に向く。

## グラフ構築

顧客をノード、全ペア間をエッジ（距離が重み）として定義する。

```python
import torch
from torch_geometric.data import Data

def build_graph(coords: torch.Tensor,
                demands: torch.Tensor) -> Data:
    n = coords.size(0)
    # 全結合エッジ
    src = torch.arange(n).repeat_interleave(n)
    dst = torch.arange(n).repeat(n)
    mask = src != dst
    edge_index = torch.stack([src[mask], dst[mask]])

    # エッジ特徴量: 距離
    diff  = coords[edge_index[0]] - coords[edge_index[1]]
    dist  = diff.norm(dim=-1, keepdim=True)

    # ノード特徴量: 正規化座標 + 需要
    x = torch.cat([coords / coords.max(),
                   demands.unsqueeze(-1)], dim=-1)
    return Data(x=x, edge_index=edge_index,
                edge_attr=dist)
```

## Attention ベースの経路デコーダ

```python
from torch_geometric.nn import GATConv

class VRPNet(torch.nn.Module):
    def __init__(self, in_f=3, h=128):
        super().__init__()
        self.conv1 = GATConv(in_f, h, edge_dim=1)
        self.conv2 = GATConv(h,   h, edge_dim=1)
        self.score = torch.nn.Linear(h, 1)

    def forward(self, data):
        x = self.conv1(data.x, data.edge_index,
                       data.edge_attr).relu()
        x = self.conv2(x, data.edge_index,
                       data.edge_attr).relu()
        return self.score(x).squeeze(-1)  # ノードスコア
```
