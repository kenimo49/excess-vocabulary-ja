# 【物流AI #1】倉庫ロボットの報酬関数設計：三層構造アプローチ

## なぜ報酬設計が難しいか

強化学習で倉庫ロボットを訓練する際、「とにかくピッキングを成功させろ」という単純な報酬だと、ロボットは商品を乱暴に掴んで破損させたり、エネルギーを浪費したりする行動を学習しやすい。報酬をどう設計するかがシステム全体の性能を左右する。

## 三層構造の報酬設計

```python
import numpy as np

def compute_reward(info: dict, action: np.ndarray) -> float:
    r = 0.0

    # ── 層1: 目標達成報酬 ──
    if info["pick_success"]:
        r += 10.0
        if info["item_damage"] == 0:
            r += 5.0  # 無破損ボーナス
        ratio = info["elapsed"] / max(info["target_time"], 1e-6)
        r += 3.0 * max(0.0, 1.0 - ratio)

    # ── 層2: 安全ペナルティ ──
    if info["item_dropped"]:
        r -= 20.0
    if info["collision"]:
        r -= 50.0
    if info["item_damage"] > 0:
        r -= 15.0 * info["item_damage"]

    # ── 層3: 効率化報酬 ──
    r -= 0.01 * float(np.sum(np.abs(action)))
    r += 0.5 * (info["prev_dist"] - info["cur_dist"])
    return r
```

## ポイント：整形報酬（Reward Shaping）

目標への距離縮小を報酬に加える「整形報酬」は学習を大幅に加速する。ただし過大にすると目標達成そのものへの動機が薄れるため、係数は 0.3〜0.5 程度が経験則だ。

## 報酬係数のチューニング指針

| 係数 | 推奨範囲 | 大きすぎると |
|---|---|---|
| ピッキング成功 | 8〜15 | 他の目標が無視される |
| 衝突ペナルティ | 30〜60 | 過度に慎重になる |
| 整形報酬 | 0.1〜0.5 | 目標未達のまま接近だけする |
| エネルギー | 0.001〜0.05 | 動かなくなる |
