# 【物流AI #7】LSTMで需要予測：在庫切れと過剰在庫を同時に減らす

## なぜ需要予測が必要か

倉庫運営では「欠品」と「過剰在庫」が収益を圧迫する。過去の出荷データから7日先の需要を予測し、発注点を動的に算出することで両方を抑えられる。

## 特徴量の設計

| 特徴量 | 説明 |
|---|---|
| 出荷数（対数変換） | 主ターゲット |
| 曜日・月 | 周期性 |
| 祝日フラグ | カレンダー効果 |
| 在庫残数 | 在庫制約 |
| 直近7日移動平均 | トレンド |

## LSTM モデル

```python
import torch, torch.nn as nn, numpy as np

class DemandLSTM(nn.Module):
    def __init__(self, feat=5, h=64, layers=2, horizon=7):
        super().__init__()
        self.lstm = nn.LSTM(feat, h, layers,
                            batch_first=True, dropout=0.2)
        self.head = nn.Linear(h, horizon)

    def forward(self, x):
        out, _ = self.lstm(x)
        return self.head(out[:, -1, :])  # (B, 7)
```

## スライドウィンドウでシーケンス生成

```python
def make_seqs(data: np.ndarray, win=30, horizon=7):
    X, y = [], []
    for i in range(len(data) - win - horizon):
        X.append(data[i:i+win])
        y.append(data[i+win:i+win+horizon, 0])
    return (np.array(X, dtype=np.float32),
            np.array(y, dtype=np.float32))
```

## 予測値から発注点を計算

```python
def reorder_point(fc7: np.ndarray,
                  lead=3, safety_z=1.65) -> float:
    mu  = fc7[:lead].mean()
    sig = fc7.std()
    return mu * lead + safety_z * sig * (lead**0.5)
```

`z=1.65` は95%サービス率に対応。リードタイム中の平均需要 + 安全在庫で発注点を決める。HuberLoss は外れ値に強くMSEより実務向き。
