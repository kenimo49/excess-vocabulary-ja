# 【物流AI #13】PatchTSTで需要予測と異常検知を同時学習する

## Transformer が時系列に強い理由

LSTM は長距離依存が苦手だが、Transformer の Self-Attention は任意の距離の依存を直接学習できる。**PatchTST** は時系列をパッチ（窓）に分割してトークン化し、効率的に学習する。

## PatchTST の実装

```python
import torch, torch.nn as nn

class PatchTST(nn.Module):
    def __init__(self, seq=336, patch=16, d=128,
                 heads=4, layers=3, horizon=7):
        super().__init__()
        n_patch = seq // patch
        self.patch  = patch
        self.embed  = nn.Linear(patch, d)
        self.pos    = nn.Parameter(torch.randn(n_patch, d))
        enc_layer   = nn.TransformerEncoderLayer(
            d_model=d, nhead=heads, dim_feedforward=d*4,
            dropout=0.1, batch_first=True)
        self.enc    = nn.TransformerEncoder(enc_layer, layers)
        self.head   = nn.Linear(d * n_patch, horizon)

    def forward(self, x):          # x: (B, T)
        B, T = x.shape
        x = x.unfold(-1, self.patch, self.patch)   # (B, P, patch)
        x = self.embed(x) + self.pos               # (B, P, d)
        x = self.enc(x).reshape(B, -1)             # (B, P*d)
        return self.head(x)                        # (B, horizon)
```

## 需要予測と異常検知の同時学習

マルチタスク学習で1つのモデルが2役をこなす。

```python
class MultitaskPatchTST(nn.Module):
    def __init__(self):
        super().__init__()
        self.backbone = PatchTST()
        fc_dim = 128 * (336 // 16)
        self.fc_fore  = nn.Linear(fc_dim, 7)   # 需要予測
        self.fc_anom  = nn.Linear(fc_dim, 1)   # 異常スコア

    def forward(self, x):
        px = x.unfold(-1, 16, 16)
        feat = self.backbone.enc(
            self.backbone.embed(px) + self.backbone.pos
        ).reshape(x.size(0), -1)
        return self.fc_fore(feat), self.fc_anom(feat).sigmoid()

def loss_fn(fp, ft, ap, at, wf=1.0, wa=0.5):
    return (wf * nn.HuberLoss()(fp, ft) +
            wa * nn.BCELoss()(ap, at))
```

## LSTM vs PatchTST 比較

| 指標 | LSTM | PatchTST |
|---|---|---|
| 長期依存（336ステップ） | △ | ◎ |
| 学習速度 | ◎ | ○ |
| 解釈可能性（Attention） | ✗ | ✓ |
| 短系列（≤60） | ◎ | ○ |

系列長が 168 ステップ（7日×1時間）以上なら PatchTST が有利。
