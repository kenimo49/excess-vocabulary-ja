# Java歴4年弱の私がFlutter(Dart)に触れて気づいた、「1つの言語をやり抜く」本当の意味

## はじめに
エンジニアになりたての頃、先輩から「**まずは1つの言語を5年続けてマスターしなさい。そうすれば他の言語の習得は容易になるから**」と言われました。当時は「そんなものか」と実感が湧きませんでしたが、最近JavaからFlutter（Dart）に挑戦したことで、その言葉の本質がようやく理解できました。

これは、かつてドイツへワーキングホリデーに行った際、初見のドイツ語を学ぶときに、学生時代に苦労して覚えた「**英語**」**というクッション**を挟んで文法を理解した感覚に非常によく似ています。

今回は、Javaという「軸」があったからこそスムーズに腑に落ちた、Dartの2つのポイントを紹介します。


## 1. 「名前付き引数」：Javaの苦い経験を解決する進化
Flutterのコードを書いていると、コンストラクタの引数が `{}` で囲まれていることに気づきます。

```dart
// Dartの名前付き引数（定義側）
const DenpyoSection({super.key, required this.data});

// 呼び出し側：ラベルを指定して渡す
DenpyoSection(data: denpyoModel);
```
Javaにはないこの書き方、実はJavaで起きがちな「引数の順番間違いバグ」を防ぐための強力なガードレールでした。

Javaでは同じ型の引数が並ぶと、順番を間違えて代入してもコンパイルエラーになりません。

```Java
// Javaの例：姓名と住所が逆でも、型がStringなら通ってしまう
public User(String firstName, String lastName, String address) { ... }
new User("東京都中央区...", "太郎", "山田"); // 実行して初めてバグに気づく
```
Dartは呼び出し側に data: というラベルを強制します。これによって、引数が多くなっても「何をどこに渡しているか」が明確になります。 「位置（順番）」ではなく「名前」で制御する——Javaという不自由さを知っていたからこそ、この設計の合理性がすぐに理解できました。

## 2. 「dynamic」：JavaのObjectと、Dartの厳格な型システム
APIレスポンスなどのMapから値を取り出す際、Dartでは dynamic 型（何でも入る型）が登場します。 ここでJavaの Object の感覚で代入しようとすると、Dartに怒られます。

```Dart
// Map<String, dynamic> から String に代入しようとしてエラーになる例
final param = DenpyoShosaiParamModel(
  denpyoNo: extra['denpyoNo'] as String? ?? '', // ← The argument type 'dynamic' can't be assigned...
);
```
「中身は文字列なんだからいいじゃないか」と思うかもしれませんが、DartはJava以上に型に厳格です。dynamic は「実行時まで中身が不明」なため、静的な型（String）に代入するには明示的なキャスト（as String）を求められます。

この「型に対する厳しさ」の意図も、Javaで静的型付けの恩恵と苦労を味わってきた経験があったからこそ、「あぁ、安全性を高めるためにここでブレーキをかけてくれているんだな」と、エラーの背景にある思想を即座に汲み取ることができました。

## 迷っている駆け出しエンジニアへの方へ：言語は「地図」になる
今、エンジニアになりたてで「この言語を学び続けていいのか？」「もっと流行りの言語があるのでは？」と迷っている人がいるかもしれません。

でも、安心してください。今学んでいる言語は、決して無駄になりません。

私がドイツ語を学んだとき、英語という「比較対象」があったからこそ、新しい文法も「英語で言うところのこれか！」と理解を早めることができました。 プログラミングも全く同じです。

1つの言語をある程度マスターすることは、単にコードが書けるようになるだけでなく、他の言語を学ぶための「地図」を手に入れることです。

**目の前の言語とじっくり向き合ってみてください**。その苦労は、数年後に新しい技術に触れたとき、「あ、これあの言語で苦労したアレの進化版だ！」という深い理解と快感に変わるはずです。

「**5年続けてみろ**」という先輩の言葉は、地図の描き方を覚えるための期間だったのだと、今なら自信を持って言えます。まだjava歴４年弱ですが。。。
