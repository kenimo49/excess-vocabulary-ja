## 1. 概要

- **対象アプリ**: プロジェクト・タスクは**同一の kintone アプリ**（例：「ガントチャート管理」）で管理
- **レコード区分**: `record_type` フィールド（`project` / `task`）でレコード種別を区別
- **設定保存先**: `kintone.plugin.app.setConfig()`
- **UI**: カンバンプラグインと同様の多タブ pqGrid ベース設定画面

## 3. ヘッダー共通機能

全タブ共通のヘッダーエリア（カンバンプラグインから流用）。

| 項目 | 種別 | 説明 |
|------|------|------|
| プラグイン有効・無効 | トグルボタン | OFF 時はカスタム一覧で何も表示しない |
| 説明 | テキストボックス | 管理者向けメモ（任意） |
| エクスポート | ボタン | 設定を JSON ファイルとしてダウンロード |
| インポート | ボタン | JSON ファイルから設定を復元 |
| 最終更新 | 表示のみ | 更新日時・更新者名を自動記録 |

## 4. タブ詳細

### Tab 1: 一覧設定

kintone アプリのカスタム一覧を取得し、ガントチャートを表示する一覧を選択・設定する。

#### グリッド列

| 列名 | 種別 | 必須 | 説明 |
|------|------|:----:|------|
| 有効 | チェックボックス | ✓ | この一覧でガントチャートを表示するか |
| 一覧名 | テキスト（読取専用） | ✓ | kintone から取得した一覧名 |
| 一覧ID | テキスト（読取専用） | ✓ | kintone の viewId |
| 備考 | テキスト | — | 管理者向けメモ |

#### 動作仕様

- 画面表示時に `kintone.app.getViews()` を呼び出し、**タイプがカスタム（type: "CUSTOM"）の一覧のみ**をグリッドに表示
- 既存設定と突き合わせ、有効フラグ・備考を復元する
- 一覧が kintone 側で削除されていた場合は行をグレーアウトして警告アイコンを表示
- 有効にできる一覧は**複数可**

---

### Tab 2: フィールドマッピング

プロジェクト・タスクが**同一アプリ**のため、このタブで両方のフィールドマッピングをまとめて設定する。
`record_type` フィールドを最上部に配置し、その下にプロジェクト・タスクのセクションを設ける。

#### 2-1. レコード区分設定（固定セクション）

| 設定項目 | 種別 | 説明 |
|----------|------|------|
| レコード区分フィールドコード | ドロップダウン | `record_type` に対応するフィールドコードを選択（DROP_DOWN型） |
| プロジェクト値 | テキスト | プロジェクトレコードを示す値（デフォルト: `project`） |
| タスク値 | テキスト | タスクレコードを示す値（デフォルト: `task`） |
