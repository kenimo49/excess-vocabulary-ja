# 【物流AI #11】転移学習とドメイン適応で新倉庫に素早く適応する

## 課題：新環境での学習コスト

新しい倉庫レイアウトやロボットを導入するたびにゼロから学習するのは時間とコストがかかる。転移学習で既存モデルを再利用し、数時間→数分で適応させる。

## Fine-tuning：最終層だけ更新

```python
import torch, torch.nn as nn

def finetune(base_model: nn.Module,
             new_act_dim: int,
             lr_body=1e-5, lr_head=1e-3):
    # 特徴抽出部を凍結
    for p in base_model.parameters():
        p.requires_grad = False

    # ヘッド（出力層）だけ新規作成
    old_h = base_model.actor.in_features
    base_model.actor = nn.Linear(old_h, new_act_dim)

    opt = torch.optim.Adam([
        {"params": base_model.trunk.parameters(),
         "lr": lr_body},          # 徐々に解凍
        {"params": base_model.actor.parameters(),
         "lr": lr_head},
    ])
    return base_model, opt
```

## DANN：ドメイン不変特徴の学習

```python
from torch.autograd import Function

class GRL(Function):
    @staticmethod
    def forward(ctx, x, alpha):
        ctx.alpha = alpha
        return x.clone()
    @staticmethod
    def backward(ctx, grad):
        return -ctx.alpha * grad, None

class DANN(nn.Module):
    def __init__(self, feat_dim, act_dim):
        super().__init__()
        self.feat   = nn.Linear(feat_dim, 128)
        self.policy = nn.Linear(128, act_dim)
        self.domain = nn.Linear(128, 2)     # src/tgt

    def forward(self, x, alpha=1.0):
        f  = self.feat(x).relu()
        pi = self.policy(f)
        d  = self.domain(GRL.apply(f, alpha))
        return pi, d
```
