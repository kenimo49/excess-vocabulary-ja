# 【物流AI #15】SimPyでデジタルツイン：倉庫シミュレーションを動かす

## デジタルツインとは

実際の倉庫をソフトウェアで再現し、AI 制御ロジックを「本番稼働前」に検証する。SimPy は Python のイベント駆動シミュレーションライブラリで、AGV の動作・搬送・待機を忠実に再現できる。

## 倉庫シミュレーションの骨格

```python
import simpy, numpy as np, dataclasses

@dataclasses.dataclass
class Config:
    n_agv: int = 4
    n_station: int = 6
    conv_speed: float = 1.5    # m/s
    sim_time: float = 28800    # 8時間

class AGV:
    def __init__(self, env, agv_id, cfg, stations, kpi):
        self.env, self.id, self.cfg = env, agv_id, cfg
        self.stations, self.kpi = stations, kpi
        env.process(self.run())

    def run(self):
        while True:
            src = np.random.randint(self.cfg.n_station)
            dst = np.random.randint(self.cfg.n_station)
            while dst == src:
                dst = np.random.randint(self.cfg.n_station)

            # 移動
            dist = abs(src - dst) * 10.0   # 仮: 10m/ステーション間
            t_move = dist / self.cfg.conv_speed
            yield self.env.timeout(t_move)

            # ステーション占有待ち
            with self.stations[dst].request() as req:
                yield req
                t_proc = np.random.exponential(30)  # 作業時間
                yield self.env.timeout(t_proc)
                self.kpi["trips"] += 1
                self.kpi["total_dist"] += dist
```

## シミュレーション実行と KPI 計測

```python
def run_sim(cfg: Config, policy=None) -> dict:
    env      = simpy.Environment()
    stations = [simpy.Resource(env, capacity=1)
                for _ in range(cfg.n_station)]
    kpi      = {"trips": 0, "total_dist": 0.0}

    for i in range(cfg.n_agv):
        AGV(env, i, cfg, stations, kpi)

    env.run(until=cfg.sim_time)
    return {
        "throughput":     kpi["trips"],
        "dist_per_trip":  kpi["total_dist"] / max(kpi["trips"],1),
        "utilization":    kpi["total_dist"] /
                          (cfg.n_agv * cfg.sim_time * cfg.conv_speed)
    }

cfg = Config()
result = run_sim(cfg)
print(result)
# {'throughput': 312, 'dist_per_trip': 42.3, 'utilization': 0.73}
```

## デジタルツインの活用シーン

- **AGV 台数の最適化**: n_agv を変えてスループット vs コストを比較
- **ボトルネック特定**: ステーション待機時間の分布を可視化
- **新レイアウト検証**: 実際に工事する前に KPI を予測
