# 【物流AI #12】TorchScript＋ROS 2でロボットにAIをエッジデプロイする

## エッジ推論が必要な理由

クラウド推論はネットワーク遅延が致命的になる。AGV や協働ロボットは **1〜10ms** のリアルタイム応答が必要なため、モデルをエッジデバイスに載せる必要がある。

## TorchScript でモデルをシリアライズ

```python
import torch

# 1. トレースで変換（動的分岐がない場合）
dummy = (torch.zeros(1,3,84,84),
         torch.zeros(1,12))          # (img, proprio)
traced = torch.jit.trace(model, dummy)
traced.save("robot_policy.pt")

# 2. スクリプトで変換（if/forがある場合）
scripted = torch.jit.script(model)
scripted.save("robot_policy_scripted.pt")
```

## 量子化で推論を高速化

```python
from torch.quantization import quantize_dynamic

q_model = quantize_dynamic(
    model,
    {torch.nn.Linear, torch.nn.LSTM},
    dtype=torch.qint8
)
print(f"元サイズ:  {param_mb(model):.1f} MB")
print(f"量子化後: {param_mb(q_model):.1f} MB")
# 例: 48.2 MB → 12.4 MB（1/4 に圧縮）
```

## ROS 2 ノードとして組み込む

```python
import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image, JointState
from std_msgs.msg import Float32MultiArray
import torch

class PolicyNode(Node):
    def __init__(self):
        super().__init__("policy_node")
        self.model = torch.jit.load("robot_policy.pt")
        self.model.eval()
        self.sub_img = self.create_subscription(
            Image, "/camera/rgb", self._cb_img, 10)
        self.sub_js  = self.create_subscription(
            JointState, "/joint_states", self._cb_js, 10)
        self.pub = self.create_publisher(
            Float32MultiArray, "/cmd_action", 10)
        self._img = self._js = None

    def _cb_img(self, msg): self._img = ros_img_to_tensor(msg)
    def _cb_js(self,  msg): self._js  = js_to_tensor(msg)

    def _publish_action(self):
        if self._img is None or self._js is None: return
        with torch.no_grad():
            mu, _, _ = self.model(self._img, self._js)
        out = Float32MultiArray()
        out.data = mu.squeeze().tolist()
        self.pub.publish(out)
```

| FP32 | 12ms | 48MB |
| INT8（TorchScript） | 3.8ms | 12MB |
