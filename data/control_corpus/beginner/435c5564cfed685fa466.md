# Electron + Drizzle ORMでローカルDBを操作するTodoアプリの作り方

## はじめに

### この記事で作るもの

この記事では、ElectronアプリケーションでDrizzle ORMを使ってローカルデータベース（SQLite）を操作するTodoアプリケーションの実装方法を解説します。

セキュリティを保ちながら、レンダラープロセスからデータベース操作を行う方法を具体的なコード例とともに紹介します。

### 想定読者

- Electronの基本的な知識がある方
- React/TypeScriptでの開発経験がある方
- ORMを使ったデータベース操作に興味がある方

## ElectronとDrizzle ORMの概要

### Electronとは

Electronは、Web技術（HTML、CSS、JavaScript）を使ってデスクトップアプリケーションを開発できるフレームワークです。VSCode、Slack、Discordなど、多くの有名なアプリケーションがElectronで開発されています。

### Drizzle ORMとは

Drizzle ORMは、TypeScript向けの軽量で型安全なORMです。SQLiteやPostgreSQL、MySQLなど様々なデータベースに対応しており、直感的なAPIと優れた型推論が特徴です。

## Electronでのファイル・DB操作の課題

### Electronのプロセスモデル

Electronアプリケーションは、大きく分けて2種類のプロセスで構成されています。

**メインプロセス**
- Node.jsの全機能にアクセス可能
- ファイルシステム、データベース操作が可能
- ウィンドウの管理を担当

**レンダラープロセス**
- ブラウザ環境で実行される
- UIの描画を担当
- セキュリティ上の理由から機能が制限されている

### セキュリティ上の制約（contextIsolation）

Electronでは、セキュリティを強化するために`contextIsolation`がデフォルトで有効になっています。これにより、レンダラープロセスとプリロードスクリプトが異なるコンテキストで実行され、直接的なアクセスが制限されます。

参考：[Electronのプロセスモデル](https://www.electronjs.org/ja/docs/latest/tutorial/process-model#%E3%83%AC%E3%83%B3%E3%83%80%E3%83%A9%E3%83%BC%E3%83%97%E3%83%AD%E3%82%BB%E3%82%B9)

### 課題：レンダラープロセスからDB操作ができない

レンダラープロセスはブラウザ環境で実行されるため、以下の制約があります。

- Node.js APIに直接アクセスできない
- ファイルシステムやデータベースを直接操作できない
- サンドボックス化されており、システムリソースへのアクセスが制限されている

これらの制約は、XSSなどのセキュリティ攻撃からアプリケーションを保護するために設計されています。

## 解決方法：IPC通信とプリロードスクリプト

### アーキテクチャの全体像

Electronでは、IPC（Inter-Process Communication）を使ってプロセス間通信を行います。以下の流れで安全にDB操作を実現します。

```
レンダラープロセス (UI層)
    ↓ window.taskStore.insertTask()
プリロードスクリプト (API公開層)
    ↓ ipcRenderer.invoke('insertTask')
メインプロセス (IPC受信 + DB操作層)
    ↓ Drizzle ORM
データベース (SQLite)
```

**レンダラープロセス**
- UIの描画とユーザー操作の処理
- `window.taskStore`経由でDB操作APIを呼び出し

**プリロードスクリプト**
- `contextBridge`を使って安全にAPIを公開
- レンダラープロセスからアクセス可能なインターフェースを定義

**メインプロセス**
- `ipcMain.handle`でプリロードスクリプトからの呼び出しを受信
- Drizzle ORMを使ってDB操作を実行

### 参考
- [Electronのプリロードスクリプト](https://www.electronjs.org/ja/docs/latest/tutorial/tutorial-preload)

### Drizzleのセットアップ

#### スキーマ定義（taskTable）

まず、データベースのスキーマを定義します。

```typescript:src/common/db/tasks.ts
import { int, integer, sqliteTable, text } from 'drizzle-orm/sqlite-core'

export const taskTable = sqliteTable('task_table', {
  id: int('id').primaryKey({ autoIncrement: true }),
  title: text('title').notNull(),
  // SQLiteにboolean型は存在しない。
  done: integer('done').notNull()
})

```

```typescript:src/common/db/schema.ts
import { taskTable } from './tasks'

export { taskTable }

```

#### マイグレーションの実装

DB初期化とマイグレーション実行の関数を用意します。

```typescript:src/main/db/common.ts
import { SqliteDatabase } from '@common/types/db.types'
import { drizzle } from 'drizzle-orm/better-sqlite3'
import { migrate } from 'drizzle-orm/better-sqlite3/migrator'

export function getDbInstance(dbPath: string) {
  try {
    return drizzle(dbPath)
  } catch (error) {
    throw new Error(`DB作成に失敗しました。\n作成に失敗したDBファイルのパス：${dbPath}`, {
      cause: error
    })
  }
}

export async function runMigrate(db: SqliteDatabase): Promise<void> {
  try {
    return migrate(db, { migrationsFolder: './drizzle' })
  } catch (error) {
    throw new Error(`DBのマイグレーションに失敗しました。`, { cause: error })
  }
}

```

型定義も用意します。

```typescript:src/common/types/db.types.ts
import Database from 'better-sqlite3'
import { BetterSQLite3Database } from 'drizzle-orm/better-sqlite3'

export type SqliteDatabase = BetterSQLite3Database<Record<string, never>> & {
  $client: Database.Database
}

```
