Anthropic Academyのクイズで間違えた。Extended Thinkingのレスポンス構造を完全に勘違いしていた。

## 2ブロック構造

Extended Thinkingを有効にすると、レスポンスの`content`配列に**2つのブロック**が返る。

```json
{
  "content": [
    {
      "type": "thinking",
      "thinking": "ステップバイステップで考えてみよう...",
      "signature": "eyJhbGciOiJFZDI1NTE5..."
    },
    {
      "type": "text",
      "text": "答えは42です。"
    }
  ]
}
```

`thinking`ブロックが先、`text`ブロックが後。

自分のコードは最後のcontentブロックだけ読んでいた。偶然動いていた。正しくは動いていなかった。

## ストリーミングの罠

ストリーミング時は`thinking_delta`イベントと`text_delta`イベントが別々に飛んでくる。

パーサーがブロックタイプを区別していないと、thinkingの生テキストがユーザー向け出力に混ざる。

## signatureは改ざん防止

`thinking`ブロックの`signature`フィールドは暗号トークン。

会話履歴にthinkingブロックを含めてClaudeに送り返す時、Claudeはsignatureを検証する。thinkingテキストが改変されていれば拒否。

なぜ？ Claudeは前回のthinkingに強く依存する。開発者がthinkingを書き換えれば、Claudeを意図しない方向に誘導できてしまう。

**実装への影響：thinkingブロックはread-only。** 会話UIでメッセージ編集機能を作るなら、thinkingブロックは編集禁止にする必要がある。

## thinkingの非表示モード

Claude 4系では`display`パラメータで思考内容の表示を制御できる。

```python
thinking={
    "type": "enabled",
    "budget_tokens": 10000,
    "display": "summarized"  # or "omitted"
}
```

`"omitted"`を指定すると、thinkingブロックのテキストは空になるが、`signature`は保持される。次のリクエストに含めれば、Claudeは前回の思考を内部的に参照できる。

**注意:** Claude 3.7 Sonnetでは`redacted_thinking`という別のブロックタイプが返ることがあった。安全システムが思考内容をフラグした場合に暗号化されるもの。モデルのバージョンによって挙動が異なるので、公式ドキュメントで確認すること。

## 設定

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,
    thinking={
        "type": "enabled",
        "budget_tokens": 10000
    },
    messages=[{"role": "user", "content": "..."}]
)
```

- `budget_tokens`は`max_tokens`より小さくなければならない
- Academyのクイズでは最小値**1024**と出題されたが、公式ドキュメントには明示されていない
- Extended Thinking有効時、temperatureは**1.0**で使用される
